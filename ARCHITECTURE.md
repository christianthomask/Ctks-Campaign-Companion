## Supabase Schema

### Phase 1 Tables

```sql
-- Users are managed by Supabase Auth (auth.users)
-- We extend with a profiles table for app-specific data

CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  display_name TEXT NOT NULL,
  role TEXT NOT NULL DEFAULT 'player' CHECK (role IN ('dm', 'player')),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE campaigns (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  description TEXT,
  dm_user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  settings JSONB DEFAULT '{}',  -- campaign-level settings
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE campaign_members (
  campaign_id UUID NOT NULL REFERENCES campaigns(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  role TEXT NOT NULL DEFAULT 'player' CHECK (role IN ('dm', 'player')),
  joined_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  PRIMARY KEY (campaign_id, user_id)
);

CREATE TABLE sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  campaign_id UUID NOT NULL REFERENCES campaigns(id) ON DELETE CASCADE,
  title TEXT NOT NULL,
  session_number INTEGER NOT NULL,
  subtitle TEXT,              -- e.g., "The Joy of Extradimensional Spaces"
  content JSONB NOT NULL,     -- structured session prep data (see data schema below)
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE session_state (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id UUID NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
  state_key TEXT NOT NULL,     -- e.g., 'puzzle_tracker', 'room_notes'
  state_value JSONB NOT NULL,  -- the mutable state data
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (session_id, state_key)
);

-- Row Level Security policies
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE campaigns ENABLE ROW LEVEL SECURITY;
ALTER TABLE campaign_members ENABLE ROW LEVEL SECURITY;
ALTER TABLE sessions ENABLE ROW LEVEL SECURITY;
ALTER TABLE session_state ENABLE ROW LEVEL SECURITY;

-- Profiles: users can read their own, DM can read all in their campaign
CREATE POLICY "Users can read own profile"
  ON profiles FOR SELECT USING (auth.uid() = id);

CREATE POLICY "Users can update own profile"
  ON profiles FOR UPDATE USING (auth.uid() = id);

-- Campaigns: DM full access, members can read
CREATE POLICY "DM full access to own campaigns"
  ON campaigns FOR ALL USING (dm_user_id = auth.uid());

CREATE POLICY "Members can read their campaigns"
  ON campaigns FOR SELECT USING (
    id IN (SELECT campaign_id FROM campaign_members WHERE user_id = auth.uid())
  );

-- Sessions: DM full access, campaign members can read
CREATE POLICY "DM full access to sessions"
  ON sessions FOR ALL USING (
    campaign_id IN (SELECT id FROM campaigns WHERE dm_user_id = auth.uid())
  );

CREATE POLICY "Members can read sessions"
  ON sessions FOR SELECT USING (
    campaign_id IN (SELECT campaign_id FROM campaign_members WHERE user_id = auth.uid())
  );

-- Session state: DM full access
CREATE POLICY "DM full access to session state"
  ON session_state FOR ALL USING (
    session_id IN (
      SELECT s.id FROM sessions s
      JOIN campaigns c ON s.campaign_id = c.id
      WHERE c.dm_user_id = auth.uid()
    )
  );
```

## Session Content Data Schema

Session prep content is stored as structured JSONB. See `/data/sessions/joy-of-extradimensional-spaces.json` for the complete example.

### Top-level structure:

```typescript
interface SessionContent {
  meta: {
    title: string;
    subtitle: string;
    level: number;
    party_size: string;
    estimated_duration: string;
    tone: string;
    lethality: string;
  };
  quick_reference: {
    setting: SettingDetails;
    npcs: NpcEntry[];
    puzzle_tracker: PuzzleEntry[];
    treasure_summary: TreasureEntry[];
    exploration_flowchart: string; // ASCII art or structured
  };
  acts: Act[];
  appendices: Appendix[];
}

interface Act {
  id: string;
  title: string;
  target_time: string;
  sections: Section[];
}

interface Section {
  id: string;              // e.g., "m7-dining-room"
  title: string;           // e.g., "M7. Dining Room"
  tags: string[];          // e.g., ["high-danger", "combat", "upper-floor"]
  read_aloud?: string;     // boxed read-aloud text
  description: string;     // DM-facing room description
  dm_notes?: DmNote[];     // callout tips
  threats?: Threat[];      // creatures in this room
  interactions?: Interaction[];  // searchable items, checks, NPCs
  treasure?: TreasureEntry[];
  puzzle_book?: PuzzleBookEntry;
  lethality_warning?: string;
  connections?: string[];  // IDs of connected rooms
}

interface Threat {
  name: string;
  stat_block: StatBlock;
  behavior: string;        // how it acts in this encounter
  combat_notes: string;    // tactical DM advice
  trigger?: string;        // what causes it to attack
}

interface StatBlock {
  name: string;
  ac: number;
  ac_note?: string;        // e.g., "natural armor"
  hp: number;
  hp_formula: string;
  speed: string;
  abilities?: {
    str: number; dex: number; con: number;
    int: number; wis: number; cha: number;
  };
  cr: string;
  attacks: Attack[];
  special_abilities?: SpecialAbility[];
  damage_vulnerabilities?: string[];
  damage_resistances?: string[];
  damage_immunities?: string[];
  condition_immunities?: string[];
  senses?: string;
  languages?: string;
}

interface Attack {
  name: string;
  to_hit: number;
  reach: string;
  damage: string;
  damage_type: string;
  effects?: string;       // additional effects on hit
}

interface SpecialAbility {
  name: string;
  description: string;
}

interface Interaction {
  action: string;          // "Search the desk", "Examine the circle"
  check?: string;          // "Investigation DC 12"
  result: string;          // what happens
}

interface DmNote {
  type: 'tip' | 'warning' | 'lore' | 'rp_guidance';
  title?: string;
  content: string;
}

interface PuzzleBookEntry {
  letter: string;
  location_description: string;
  discovery: string;       // how to find it
  illustration: string;    // what the breadcrumb illustration shows
}

interface NpcEntry {
  name: string;
  location: string;
  voice: string;
  key_info: string;
}

interface PuzzleEntry {
  letter: string;
  room: string;
  found: boolean;         // mutable state, tracked in session_state
}

interface TreasureEntry {
  room: string;
  item: string;
  value: string;
}
```

## Component Rendering Logic

The `SessionViewer` component is the Phase 1 workhorse. It:

1. Fetches session content JSON from Supabase
2. Renders the content as a scrollable document with collapsible sections
3. Each section type renders via a specialized component:
   - `ReadAloud` — styled box with serif font, distinct background, quotation marks
   - `StatBlock` — collapsed by default, tap to expand, game-stat styling (monospace numbers, attack formatting)
   - `DmNote` — colored callout box (blue for tips, red for warnings, purple for lore, green for RP guidance)
   - `LethalityWarning` — red-bordered callout with ⚠️ icon
   - `PuzzleTracker` — interactive checkboxes, persisted to session_state
   - `NpcTable` / `TreasureTable` — compact reference tables
4. `SearchOverlay` provides full-text search across all section titles, descriptions, NPC names, creature names, and read-aloud text
5. Navigation via a collapsible sidebar (desktop) or bottom sheet (mobile) listing all sections as jumpable links

## 5e.tools Data Integration (Phase 4)

5e.tools publishes its data as JSON on GitHub: `https://github.com/5etools-mirror-3/5etools-src`

Key data files:
- `data/spells/` — spell data by source book
- `data/bestiary/` — monster data by source book
- `data/items/` — item data
- `data/conditionsdiseases.json` — conditions and diseases
- `data/actions.json` — actions in combat

**Integration approach:**
1. Write a seed script that fetches the JSON files from GitHub
2. Parse and normalize into our Supabase table schemas
3. Run once to populate, re-run to update when 5e.tools updates
4. Add full-text search indexes in Postgres for fast lookups

This is Phase 4 work — don't build it now, but the schema is designed to accommodate it.
